# プレゼンテーション設計フレームワーク

## 聴衆別アプローチ

### 経営層向けプレゼン
- **結論先行型**: 最初に結論と効果を明示
- **数値・ROI重視**: 具体的な数字とビジネスインパクト
- **時間厳守**: 予定時間の90%で完了
- **意思決定支援**: 明確な選択肢と推奨事項

### 技術者向けプレゼン
- **詳細説明重視**: 技術的根拠と実装方法
- **実装可能性**: 現実的な技術選択と制約
- **データ重視**: 性能数値、比較データ
- **質疑応答充実**: 技術的議論の時間確保

### 一般聴衆向けプレゼン
- **わかりやすさ**: 専門用語の回避、平易な表現
- **具体例多用**: 身近な例、比喩の活用
- **親しみやすさ**: ストーリー性、感情に訴える要素
- **視覚的支援**: 図表、画像の効果的使用

## 構成パターン

### PREP法（論理的説明）
1. **Point（結論）**: 最初に結論を明示
2. **Reason（理由）**: なぜその結論に至るか
3. **Example（例）**: 具体的な事例や証拠
4. **Point（結論再述）**: 結論を再度強調

### 問題解決型（提案・改善）
1. **問題提起**: 現在の課題や問題点
2. **原因分析**: 問題の根本原因
3. **解決策提示**: 具体的な改善案
4. **効果説明**: 解決による効果・メリット

### ストーリー型（体験・事例共有）
1. **現状説明**: 取り組み前の状況
2. **課題発見**: 直面した困難や課題
3. **取り組み内容**: 実施した対策や努力
4. **成果・学び**: 得られた結果と教訓

## 時間別構成ガイド

### 5分プレゼン
- **導入**: 30秒（テーマと結論）
- **本論**: 3分（要点1つ + 根拠）
- **結論**: 1分（まとめと次のアクション）
- **余白**: 30秒（質問対応）

### 15分プレゼン
- **導入**: 2分（背景とアジェンダ）
- **本論**: 10分（要点3つ、各3分強）
- **結論**: 2分（まとめと提案）
- **質疑**: 1分（簡単な質問対応）

### 30分プレゼン
- **導入**: 3分（背景、目的、アジェンダ）
- **本論**: 20分（詳細説明、複数論点）
- **結論**: 4分（総括と提案）
- **質疑**: 3分（充実した議論時間）

## スライド設計原則

### 視覚的原則
- **1スライド1メッセージ**: 伝えたいことを1つに絞る
- **6×6ルール**: 1行6語以内、1スライド6行以内
- **コントラスト**: 重要な部分を強調
- **一貫性**: フォント、色使いの統一

### 内容設計
- **タイトルの明確化**: 各スライドの主張を明示
- **論理的流れ**: 前後のつながりを意識
- **証拠の提示**: 主張の根拠となるデータ
- **アクション指向**: 聴衆に求める行動の明示

## 発表スキル

### 話し方のコツ
- **間を活用**: 重要なポイント前後で一呼吸
- **声のトーン**: 単調にならない抑揚
- **アイコンタクト**: 聴衆全体を見渡す
- **ジェスチャー**: 自然な身振り手振り

### 準備のポイント
- **リハーサル**: 本番の8割の速度で練習
- **想定質問**: 予想される質問と回答準備
- **技術確認**: 機材、接続の事前チェック
- **バックアッププラン**: トラブル時の代替案
